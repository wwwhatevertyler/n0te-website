## What N0te Is

N0te is a tiny floating capture layer for macOS that sends clean Markdown into Obsidian without pulling the user out of flow.

Internal north star:

> Obsidian is where thoughts become a system. N0te is where thoughts survive the moment.

The product is a calm, premium Mac utility for catching fragments before they disappear, then routing them into a durable local system the user already trusts.

## What N0te Is Not

N0te is not:

- a full notes app
- a task manager
- a second Obsidian
- a writing workspace
- a generic sticky-note clone
- an AI wrapper or chat client
- a full PKM platform

The website should keep the product narrow and sharp. The value is not “more features.” The value is fast local capture without context switching.

## Who It Is For

Primary audience:

- Obsidian users who want faster capture on Mac
- designers, builders, founders, researchers, writers, and operators who collect fragments constantly
- AI-assisted builders who need a durable place for prompts, bugs, specs, and decisions outside the chat window

These users care about:

- speed
- staying in the current app
- local files
- clean Markdown
- low friction before organization
- a tool that feels calm enough to leave open

## Core Promise

Public-safe promise:

> Capture before the thought disappears.

Supporting explanation:

> N0te is a floating note for macOS that sends clean Markdown into Obsidian without pulling you out of flow.

What the product protects is not storage. It protects momentum.

## Current Feature Set The Website Can Talk About

Safe current features:

- floating macOS note windows
- local-first note storage
- multiple independent notes
- clean Markdown export into an Obsidian vault
- separate image attachments with export support
- Apple Speech-backed dictation capture
- first-run onboarding
- per-note light and dark mode
- polished tactile hover and press interactions

Website-safe phrasing:

- floating note for macOS
- clean Markdown to Obsidian
- local-first capture
- keep a thought in reach
- capture ideas, prompts, bugs, and reference fragments

Do not imply features that are not ready as shipped expectations, including:

- global hotkey
- deep routing presets beyond the current Obsidian export path
- cross-device sync
- advanced automation
- direct AI integrations

## Current Maturity Level

N0te is in active product development. It already has a strong core interaction model and a polished visual identity, but it should still be described as an early premium Mac utility rather than a fully mature workflow platform.

Important maturity framing:

- strong capture experience is real
- Obsidian export is real
- visual polish is real
- the paid-utility story is credible
- some “must ship before charging” infrastructure is still ahead, especially global hotkey, stronger Obsidian routing, and trust/completion details around packaging and launch readiness

## Proof Points

Current proof points the website can lean on:

- native macOS product, not a browser wrapper
- visually distinctive floating glass note object
- clean local Markdown exit into Obsidian
- can stay visible above work so capture happens in context
- small enough to stay out of the way, polished enough to keep open
- useful for project memory as well as personal thought capture

## Product Constraints For Website Claims

The website should not require readers to understand the app architecture. It should explain the user outcome.

Translate implementation into outcomes:

- “manually managed windows” becomes “floating notes that stay within reach”
- “plain-text NSTextView” becomes “clean text capture with a clean Markdown exit”
- “separate image attachments” becomes “keep screenshots and files with the note without turning the note into a cluttered canvas”

Do not market hidden implementation details as product value unless they clearly matter to the user.
